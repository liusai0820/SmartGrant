# app/ 架构说明

> ⚠️ **自指声明**：一旦我所属的文件夹有所变化，请更新我。

## 极简架构（3行）
- **page.tsx** - 主应用入口，渲染完整的评审工作台界面
- **api/** - 后端 API Routes，处理 AI 调用、文件上传、数据持久化
- **layout.tsx + globals.css** - 全局布局与样式，定义应用骨架

---

## 文件清单

| 名称 | 地位 | 功能 |
|------|------|------|
| `page.tsx` | 🏠 主入口 | 主页面组件，编排所有子组件形成完整工作台 |
| `layout.tsx` | 🎨 布局 | 根布局，注入字体、主题 Provider、元数据 |
| `globals.css` | 🎨 样式 | 全局 CSS 变量与 Tailwind 配置 |
| `api/` | 📡 API层 | 所有后端 API Routes 的父目录 |

---

## api/ 子目录清单

| 名称 | 功能 |
|------|------|
| `chat/` | AI 对话接口，支持基于项目上下文的智能问答 |
| `expert/` | 专家遴选接口，根据项目技术领域匹配专家 |
| `export/` | 文档导出接口，生成 Word 公文 |
| `extract-metadata/` | AI 元数据提取，自动识别项目名称、单位等 |
| `review/` | 核心评审接口，调用多 Agent 协同评审 |
| `upload/` | 文件上传接口，处理 PDF/Word 解析 |
